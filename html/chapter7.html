第7章：GPU 架构演进

从固定功能管线到可编程并行处理器的技术革命

本章概览

NVIDIA GPU架构的演进历程是现代计算史上最重要的技术革新之一。从1999年GeForce 256定义GPU概念开始，经过25年的发展，GPU已经从专用图形处理器演变为通用并行计算平台，成为AI时代的核心基础设施。

本章将深入剖析三个核心技术维度：

  1. SM（Streaming Multiprocessor）演化史：从固定功能单元到高度可编程的并行处理器
  2. 内存架构革新：从DDR到HBM，解决"内存墙"问题的技术演进
  3. 互连技术发展：NVLink和NVSwitch如何打破PCIe瓶颈，实现超大规模GPU集群

1. SM (Streaming Multiprocessor) 演化史

1.1 前GPU时代：固定功能管线 (1995-1999)

在GPU概念诞生之前，图形加速器采用固定功能管线架构。这个时期的设计理念是用专用硬件加速特定的图形操作，每个功能都由独立的电路实现。

┌──────────────────────────────────────────────────┐
│            NV1/RIVA 128 固定管线                  │
├──────────────────────────────────────────────────┤
│  顶点处理 → 光栅化 → 纹理映射 → 像素输出          │
│     ↓         ↓         ↓          ↓            │
│  [固定单元] [固定单元] [固定单元] [固定单元]      │
└──────────────────────────────────────────────────┘

NV1 (1995) 的独特设计：

  • 采用二次曲面渲染（Quadratic surfaces）而非三角形
  • 与世嘉Saturn游戏机架构对齐
  • 音频处理集成（SoundBlaster兼容）
  • 最终因与业界标准（Direct3D）不兼容而失败
  • 教训：技术创新必须考虑生态兼容性

RIVA 128 (1997) 的成功转型：

  • 完全支持Direct3D和OpenGL标准
  • 128位内存接口（业界领先）
  • 硬件加速的透视校正纹理映射
  • 双线性和三线性过滤
  • 每秒100万多边形处理能力
  • 月销量超过100万片，拯救了NVIDIA

RIVA TNT/TNT2 (1998-1999)：

  • 32位真彩色渲染（竞品仅16位）
  • 双纹理单元设计
  • AGP 2X/4X支持
  • 250MHz核心频率（TNT2 Ultra）
  • 与3dfx Voodoo2正面竞争并最终获胜

技术特征：

  • 每个处理阶段都是专用硬件单元
  • 无法编程，只能通过固定API控制
  • 效率高但灵活性极差
  • 无法适应新的渲染算法
  • 硬件利用率低：某些阶段空闲时其他阶段也无法使用其资源

1.2 第一代GPU：硬件T&L时代 (1999-2001)

GeForce 256 (NV10)：定义GPU概念，引入硬件T&L（Transform & Lighting）

1999年8月31日，NVIDIA发布GeForce 256，首次提出GPU（Graphics Processing Unit）概念。这不仅是营销术语的创新，更代表了图形处理架构的根本性变革。

┌─────────────────────────────────────────────────────┐
│               GeForce 256 架构                       │
├─────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌──────────┐  ┌─────────┐           │
│  │ T&L单元  │→│ 光栅化器  │→│ 像素管线 │           │
│  │(硬件固定)│  │          │  │(4条并行) │           │
│  └─────────┘  └──────────┘  └─────────┘           │
│                                                     │
│  关键参数：                                          │
│  • 23M 晶体管 (修正值)                               │
│  • 220nm 制程 (TSMC)                                │
│  • 480M 顶点/秒                                     │
│  • 4 像素管线                                       │
│  • 120MHz 核心频率                                  │
│  • 166MHz DDR内存                                   │
└─────────────────────────────────────────────────────┘

硬件T&L的革命性意义：

  • CPU解放：将几何变换从CPU转移到GPU，CPU利用率降低50%
  • 性能飞跃：比软件T&L快4-10倍
  • 光照计算：硬件支持8个光源的实时计算
  • 立方环境映射：实现实时反射效果
  • 顶点混合：支持骨骼动画（4个矩阵）

GeForce2 GTS (NV15, 2000)：

  • 第二代T&L引擎
  • 引入像素着色器前身NSR（NVIDIA Shading Rasterizer）
  • 每时钟4个纹理采样
  • 1.6 GigaTexels/秒填充率
  • 首次支持S3TC纹理压缩

市场影响：

  • 彻底击败3dfx Voodoo3（无硬件T&L）
  • 确立NVIDIA在高端市场的领导地位
  • 推动DirectX 7.0标准的普及
  • 游戏开发者开始为硬件T&L优化

3. 互连技术发展（NVLink、NVSwitch）

随着AI大模型的兴起，单个GPU的算力已经无法满足需求。多GPU并行成为必然选择，而互连技术成为决定系统性能的关键。NVIDIA通过NVLink和NVSwitch技术，构建了业界最强大的GPU互连网络。

3.1 PCIe瓶颈与突破

PCIe的历史局限性： PCIe作为通用总线标准，虽然广泛应用，但其带宽增长速度远远落后于GPU内部带宽的发展。这种不匹配成为多GPU系统的最大障碍。

5. 架构设计哲学演变

NVIDIA GPU架构的演变不仅是技术的进步，更是设计哲学的演化。每一次重大架构变革都反映了对计算本质的更深入理解。

5.1 从专用到通用的转变

GPU发展历程中最重要的转变是从图形专用处理器到通用计算平台的过渡。这个转变不是一蹴而就的，而是经历了多个阶段的渐进演化。

第一阶段 (1999-2006): 图形专用
• 固定功能管线
• 顶点/像素分离处理
• 优化光栅化性能

第二阶段 (2006-2016): 通用计算觉醒
• 统一着色器架构
• CUDA编程模型
• 科学计算应用

第三阶段 (2017-至今): AI专用加速
• Tensor Core矩阵运算
• 混合精度训练
• Transformer专用引擎

5.2 架构设计权衡

现代GPU架构设计需要在多个维度之间进行精巧的平衡：通用性vs专用性、性能vs功耗、复杂度vs可编程性。每一代架构都代表了在特定技术条件下的最优权衡。

本章总结

NVIDIA GPU架构的25年演进历程是现代计算技术发展的缩影。从固定功能管线到可编程着色器，从统一架构到Tensor Core，每一次技术飞跃都精准地捕捉到了时代的需求。

NVIDIA GPU架构的演进历程展现了计算平台设计的范式转变：

  1. 架构创新驱动力：从图形渲染需求到通用计算，再到AI训练，每个时代的应用需求都推动着架构的革新。

  2. 系统级优化：现代GPU不再是单一处理器，而是包含计算、内存、互连的完整系统，每个组件都经过精心优化。

  3. 软硬件协同设计：CUDA生态系统与硬件架构的紧密结合，创造了难以复制的技术壁垒。

  4. 前瞻性布局：NVIDIA在2006年推出CUDA、2017年引入Tensor Core的决策，都远早于市场需求爆发，展现了卓越的技术远见。

GPU架构的演进还在继续，随着AI大模型、元宇宙、量子计算等新兴应用的出现，未来的GPU架构必将迎来更多革命性创新。
